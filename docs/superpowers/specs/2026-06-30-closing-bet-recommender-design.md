## 0. 한 줄 요약

매 거래일 **15:20**(정규장 마감 15:30 직전)에, "종가베팅(신접갈거조재 + 시황 + 수급)" 방법론을 계산 가능한 신호로 번역하여 **다음날 상승 가능성이 높은 종목을 점수 순으로 최대 30개** 추천하는 대시보드. 오버나잇 효과(종가 매수 → 익일 시초/장 초반 매도)를 캡처한다. 자동 주문은 하지 않으며, **베테랑의 판단을 돕는 의사결정 지원 도구**로 설계한다.

## 1. 목표 / 비목표

### 목표
- 15:20 스냅샷으로 익일 상승 후보를 랭킹(최대 30개)하여 매수가/목표가/손절가/차트와 함께 제시
- 매일 추천을 자동 로깅하고 **익일 결과로 채점하여 적중률(성공률)** 추적
- 시황·수급·과열·희석 등 리스크 오버레이로 "사면 안 되는 날/종목"을 걸러냄

### 비목표 (이번 범위 아님)
- 자동 주문/체결 (toss-autotrade가 담당, 책임 분리)
- 실시간 인트라데이 트레이딩(스캘핑) 신호
- 파생/해외/암호화폐
- 9개 신호 전부의 정밀 점수화 (YAGNI — 1차는 핵심만)

## 2. ⚠️ 전제와 솔직한 리스크 (설계 의사결정의 토대)

적대적 검증에서 확인된 핵심 — **반드시 기대치 설정에 반영한다.**

1. **9개 조건 → 실제 정보축은 3개.** 신·접·갈·조·거·재(긍정부)는 "오늘 거래량 터지며 고가 부근 강마감한 유동성 좋은 종목"이라는 **하나의 베팅의 다른 옷**. 따라서 점수모델은 이 셋을 **곱셈형 확인(confirmation)** 으로 묶어 실효적으로 **3축(① 돌파강도 ② 시황 레짐 ③ 수급)** 만 남긴다(§3.3). 더하기(additive)는 직교 축에만 허용한다.
2. **단기 반전과 싸운다.** 종가베팅 픽은 한국(특히 코스닥)에서 가장 강건한 단기 효과인 **단기 반전(short-term reversal)** 에 역행한다. 인용 근거(52주 신고가=George-Hwang 2004, 거래량 프리미엄=Gervais-Kaniel-Mingelgrin)는 전부 **월간** 횡단면 효과이며, **1일 오버나잇** 으로의 이식은 **미검증**. 매도세(0.18%)+호가스프레드+단일가 슬리피지 차감 후 순엣지는 ≤0일 수 있다.
3. **진짜 가치는 종목 찍기가 아니라 리스크 오버레이 + 청산 규율.** 시황 게이트, 희석 veto, 유동성/과열 가드, "보유 금지(익일 시초/초반 청산)" 규율이 좌측꼬리를 줄인다.
4. **신뢰 전 입증 의무(수치 기준 명시).** 어떤 신호든 **walk-forward rank-IC > 0 (t-stat > 2) 이고 순비용 차감 incremental IC가 오버나잇-ETF/무작위 베이스라인을 상회** 하기 전에는 어떤 가중치/배수 범위도 키우지 않는다.

> 결론: "알파 자동 발굴기"가 아니라 "검증된 리스크 필터 위에서 후보를 랭킹·시각화하는 의사결정 지원 도구". 백테스트는 낙관 편향(생존편향·룩어헤드·잠정→확정 갭)을 적극적으로 제거한 뒤에만 신뢰한다.

## 3. 전략 → 계산 가능한 신호 (확정 MVP 셋)

### 3.1 채택/제외 요약

| 원 조건 | 처리 | MVP |
|---|---|---|
| **시황 (지수 5일선 레짐)** | HARD 게이트, 종목 소속시장별 적용 (§3.2-A) | ✅ |
| **신 (52주 신고가 근접도)** | 돌파강도 **base** (연속 nearness, [예측근거: 강함(월간)/오버나잇 미검증]) | ✅ |
| **거 (거래대금/거래량)** | 후보 풀 스크리너 + 돌파의 **RVOL 확인 배수** (additive 아님) | ✅ |
| **수급 (외인·기관)** | **D-1 확정치** 양방향 부스터(0.8~1.2x) | ✅ |
| **재 (재료)** | **희석성 공시 = 매수 veto** (긍정 재료 점수화 제외) | ✅(veto만) |
| 위생 가드 | 유동성·우선주/ETF·관리/경고·과열(상한가/VI)·거래정지 제외 | ✅ |
| 접 (전고점 이격) | 신과 collinear → **차트에 선만** | 시각만 |
| 갈 (깔끔한 양봉) | 캔들 알파 약함 → **차트 시각만** | 시각만 |
| 조 (조정 기간) | 9파라 트리거 과최적 → "충분히 조정됨" **베이스 배지(시각)** | 시각만 |
| 섹터 주도성 | 1일 edge 얇음 → 2차 | 2차 |

> **MVP 가중합에는 s_접·s_갈·s_조가 들어가지 않는다.** (조는 차트 배지로만 노출)

### 3.2 신호별 정의 (계산 규칙)

**A. 시황 — 지수 5일선 레짐 게이트** [근거: 강건한 추세필터, 알파 아님]
- 코스피·코스닥 **각각** 산출. 입력: 지수 종가(15:20 잠정) + 5일 이평(전일까지 4항 확정 + 당일 잠정 1항).
- **Truth table** → `regime_mult`:

  | 조건 | regime_mult |
  |---|---|
  | 지수 ≥ 5일선 **그리고** 5일선 상승(또는 당일 반등) | **1.0** |
  | 지수 < 5일선 **그러나** 당일 5일선 위로 회복/반등(당일 등락률>0) | **0.5** |
  | 지수 < 5일선 **그리고** 하락(5일선 하락) | **0.0 (중단)** |

- **적용 규칙**: 종목은 **자신의 상장 시장**(KOSPI/KOSDAQ) 레짐만 사용. 두 시장 모두 0이면 추천 보드는 빈 상태 + RISK_OFF 배너(§4.1). regime_mult=0 시 해당 시장 종목은 final=0으로 제외.
- 인덱스 코드 충돌 주의: pykrx `1001`=KOSPI / `2001`=KOSDAQ vs KIS `0001`=KOSPI / `1001`=KOSDAQ. **중앙 매핑(BrokerDataAdapter)** + KOSPI 동일성 단위테스트로 방지.

**B. 신 — 52주 신고가 근접도 (George-Hwang nearness)** [근거: 강함(월간)/오버나잇 미검증 → 잠정 기본 비중 중립]
- `P_now` = 15:20 잠정 스냅샷가(KIS). `H_ref` = **전일까지 확정 EOD 일봉 고가**의 롤링 최대(룩어헤드 금지).
- 두 창: N1=252거래일(52주), N2=60일(~3개월).
- `s_신 = 0.7·clip((near_252−0.90)/0.10,0,1) + 0.3·clip((near_60−0.90)/0.10,0,1)`, `near_X = P_now / max(High[t-X..t-1])`.
- **이력 요건**: near_252는 **이력 ≥252거래일** 일 때만 계산. 120~251일 종목은 **near_60만 사용**하고 라벨을 "가용구간 고가(available-window high)"로 표기(252일 신고가로 오인 금지). 120일 미만 제외.
- **과열 가드**: 당일 등락률 ≥ +20% 또는 상한가/VI 발동 → buy로 emit 금지.
- "신고가 돌파 ON CLOSE"는 15:20~15:30 단일가에서 뒤집힐 수 있으므로 **provisional 플래그**; 연속 nearness 비율을 신뢰.

**C. 거 — 거래대금/RVOL (돌파 확인 배수, additive 아님)** [근거: 보통, 비용취약]
- 후보 풀 = §3.3 정의(D-1 상위 ∪ 라이브 top-30×2).
- **시간대 정규화 RVOL**: `RVOL = (당일 15:20 누적거래량) / (20일 평균의 "15:20 시점 누적" 추정)` — 부분장 대 전일종일 비교의 미스캘리브레이션 제거(백테스트도 동일 부분장 정규화로 미러).
- `s_거 = clip(log2(RVOL)/log2(3), 0, 1)`.
- **확인 배수**: `rvol_confirm = clip(0.6 + 0.4·s_거, 0.6, 1.0)` — 거래량 빈약한 돌파는 할인, 충분하면 만점. (거를 독립 가산점으로 두지 않음으로써 신–거 collinearity 차단)

**D. 수급 — 외인·기관 (D-1 확정, 양방향)** [근거: 진짜·직교, modest]
- pykrx `get_market_net_purchases_of_equities(fromdate, todate, market, investor)` — **전 종목 일괄 1회**로 D-1 외인/기관 순매수액. (per-ticker 반복 호출 금지)
- 정규화 `z = clip((외인+기관 D-1 순매수액) / (20일 평균 거래대금), -1, 1)`.
- **양방향 부스터**: `supply_tilt = clip(1.0 + 0.2·z, 0.8, 1.2)` — D-1 순매도(분산)는 1.0 미만 패널티, 순매수는 부스터.

**E. 재 — 희석 veto (DART 공시)** [근거: veto가 핵심 가치]
- **윈도우**: 확정 공시 **T-1 15:20 ~ T 15:20** 스캔(장중 공시는 실시간이라 사용 가능). post-15:30 당일 공시는 §5 POST-CLOSE 함정 → 채점/추천에서 제외.
- **희석 화이트리스트(보고서명)**: 유상증자결정, 전환사채권발행결정, 신주인수권부사채권발행결정, 교환사채권발행결정 → `veto=0`(추천 제외). **무상증자/주식배당은 제외(non-dilutive, false-veto 금지).**
- corp_code(8자리)↔티커(6자리) 매핑(corpCode.xml) 주기적 갱신, 미매핑 시 폴백.
- **구조적 한계 명시(UI/기대치)**: 15:20 veto는 **post-15:20 당일 공시(장 마감후 유상증자 등 "오버나잇 폭탄")** 를 볼 수 없다. **익일 아침 재스캔**으로 해당 종목을 성과 로그에 플래그(이미 확정된 추천을 바꾸진 않음).

**F. 위생 가드** — 20일 평균 거래대금 ≥ **10억(글로벌 유동성 바닥, 1회 정의)**, 우선주/ETF/ETN/SPAC 제외, 관리·투자경고·투자위험·단기과열 지정 제외, 과열(상한가/등락률≥+20%/VI) 제외, **15:20 거래정지 종목 제외**.

### 3.3 복합 점수 (MVP 실효 공식, 곱셈형 — 실효 3축)
```
# 후보 풀
pool = (pykrx D-1 거래대금 상위 N[기본 200], FINAL·재현가능)   # 장전 사전조회
     ∪ (KIS 라이브 top-30 KOSPI + top-30 KOSDAQ @15:20)        # 당일 신선 돌파 포착
# 위생 필터(§3.2-F) 통과분에 대해서만 15:20 라이브 가격/거래량 조회(~260종목)

breakout    = s_신                              # MVP: 접/갈/조 가산 없음(시각만)
rvol_confirm= clip(0.6 + 0.4·s_거, 0.6, 1.0)     # 거 = 돌파 확인(곱)
final       = breakout × rvol_confirm × regime_mult(0/0.5/1) × supply_tilt(0.8~1.2) × veto(0/1)
```
- **emit 규칙**: `final > 0` 인 종목만, **점수 내림차순 상위 30**. 동점 tie-break = D-1 거래대금 큰 순. 통과 0개면 빈 보드 + 사유 배너.
- **가중치/배수 범위/임계값은 §2.4 입증 기준 충족 전 잠금.** (지금은 잠정 기본값)

### 3.4 가격 규칙 (행마다, 결정론적)
- **현재가(표시)** = 15:20 잠정 스냅샷(KIS), "잠정" 워터마크.
- **매수가(체결 기준)** = **15:30 종가단일가**(익일 아침 pykrx로 확정). 표시는 15:20 잠정, **채점·백테스트는 확정 종가**(§4.2).
- **청산(주 행동 CTA)** = **익일 시초가 / 09:30 VWAP** ("보유 금지 권장").
- **목표가/손절가(참고용, '보유 시'로 라벨)** — 결정론:
  - 손절 = `max(매수가 − 1.0·ATR20, 매수가·0.97)` (둘 중 타이트한 값)
  - 목표 = `min(매수가 + 1.2·ATR20, 직전 전고점)` 단 **전고점 ≤ 매수가면 ATR 가지 사용**(nearness≥1.0 종목 보호)
  - k=1.2 고정. ATR20은 일간 레인지라 오버나잇 호라이즌에선 거의 트리거되지 않음 → **참고 정보**임을 UI에 명시.

## 4. 추천 산출물 & 성과 추적

### 4.1 추천 보드 (메인)
- 점수 내림차순 **최대 30개**. 각 행: 종목명/코드, 현재가(잠정), 매수가, **청산(주 CTA)**, 목표가/손절가(참고), 점수/**등급(S/A/B/C)**, 신호 배지(신고가·RVOL·수급·시황·베이스·잠정), 미니 차트.
- **등급 컷오프**: final 점수 고정 임계(예: S≥0.8, A≥0.6, B≥0.4, C>0) — §4.2 점수구간별 적중률의 버킷과 동일.
- **상단 강조 top-3~5**: 10분(15:20~15:30) 내 즉시 행동용. 실행 완료 시 데스크톱/푸시 알림.
- **빈 보드 상태**: RISK_OFF/통과 0개 시 "오늘은 시황 레짐상 추천 없음" 배너(시황 게이지 연동) — 버그가 아닌 의도된 리스크 결정임을 명시.

### 4.2 성과 추적 (적중률)
- **성공 정의(확정)**: `open[t+1] / close[t] − 1 > 0` — **체결 가능한 15:30 확정 종가 기준**(15:20 잠정은 표시/진단용 보조 지표로만 별도 표기). 라이브·백테스트 **동일 기준** → 직접 비교 가능.
- **익일 시가 결측/잠김 처리**: 익일 시가가 상한가 잠김(체결 불가)·거래정지·데이터 없음 → **N/A 처리, 적중률 분모에서 제외**(0점 처리 금지).
- 매일 15:20 추천 스냅샷 + 익일 확정 종가/시가 저장 → 자동 채점.
- 지표: 어제 N개 중 성공 M개(적중률 %), 평균 익일 시초 수익률, 누적 곡선, **점수구간(등급)별·시황레짐별 적중률**.
- **콜드스타트**: 표본 누적 전(예: 누적 픽 < 30)에는 "데이터 누적 중" 플레이스홀더 + 최소표본 캡션, 적중률 회색 처리(초기 노이즈를 검증된 성과로 오인 방지).

## 5. 15:20 데이터 현실 (FINAL/PROVISIONAL/POST-CLOSE)

| 구분 | 항목 | 처리 |
|---|---|---|
| **FINAL(확정)** | 전일까지 EOD OHLCV·고가·20일평균(거래대금/거래량), **D-1 외인/기관 순매수**, 시황 5일선 4/5항, 지수 이력 | **장전 사전조회·캐시** (15:20 임계경로에서 제외) |
| **PROVISIONAL(잠정)** | 당일 현재가·누적거래량·지수레벨(5번째 항)·과열상태 | **KIS 인트라데이 피드** (pykrx는 EOD 전용) |
| **POST-CLOSE ONLY(함정)** | 당일 공식 외인/기관 순매수, **post-15:30 공시** | **자동 점수화 금지** — 익일 재스캔/보조 큐로만 |

- **회피책**: 수급은 D-1 확정만 점수화. 매수가 채점은 15:30 확정 종가. DART veto는 15:20까지 확정 공시만(장중 실시간 사용). provisional 워터마크 + 15:20→15:30 픽 변동 가능성 UI 명시.

### 6.2 일일 흐름
```
[장전] FINAL 사전조회·캐시: H_ref/20일평균/D-1 수급/지수 이력 + 헬스체크(최근 거래일·행수 검증, 실패시 fail-closed)
15:20 스케줄러
  ① 후보 풀: pykrx D-1 거래대금 상위 200(캐시) ∪ KIS 라이브 top-30×2
  ② 위생 필터: 유동성·종목타입·관리/경고·과열·거래정지 제외
  ③ 라이브 조회: 통과분(~260)만 KIS 현재가/누적거래량 + 지수/과열상태
  ④ 시황 게이트(종목별 시장 레짐) → RISK_OFF 시장 제외
  ⑤ 신호 계산: 신(near)·거(rvol_confirm)·수급(supply_tilt)·재 veto
  ⑥ final 정렬 → 상위 30 → 가격규칙(매수가=잠정표시, 청산 CTA, 참고 목표/손절)
  ⑦ 스냅샷 저장 + 완료 알림(top-3 강조)
[익일] 확정 종가/시가로 어제 추천 자동 채점 + DART 오버나잇 공시 재스캔 → 적중률 갱신
```

## 7. 백테스트 & 검증

- **신호가 재현**: 가능하면 인트라데이 이력으로 **15:20-등가 스냅샷 재구성**하여 신호 산출. 불가 구간은 **측정된 haircut + 민감도 밴드**로 EOD→15:20 신호 드리프트를 정량화(각주가 아니라 수치로 bound).
- **진입/채점가**: 진입·채점 모두 **확정 종가 close[t]** 기준(라이브와 동일) → 라이브·백테스트 비교 가능.
- **풀 재구성**: 라이브 풀(D-1 상위 ∪ 라이브 top-30×2)을 백테스트에서도 **D-1 상위로 시점기준 재현**(전일 데이터라 룩어헤드 없음). 잔여 스큐 문서화.
- **생존편향 제거**: point-in-time 유니버스 = pykrx `get_market_ticker_list(date)` + **KRX 상장폐지/정리매매 리스트(유지 스냅샷)**. 소스 확정 전엔 생존편향 제거 주장 downscope.
- **분할 보정 범위(축소)**: nearness/가격 비율은 일관 보정이면 robust(저위험). **RVOL/거래량**에 집중(분할 시 거래량 점프) → split-consistent 거래량 또는 거래대금(분할 불변) 사용.
- **수용 기준(go/no-go)**: walk-forward rank-IC > 0 (t > 2) **그리고** 순비용 차감 incremental IC > 오버나잇-ETF/무작위 베이스라인. 미충족 시 가중치/배수 범위 상향 금지.

## 8. 안전 & 테스트

### 안전/정합성
- 추천 전용(주문 없음) → 자금 리스크 없음. 모든 15:20 산출물 provisional 표기.
- 룩어헤드 방지(H_ref 전일까지·EOD 프록시 라이브 금지), 생존편향 제거(§7), 잠정→확정 갭 정량화.

### 테스트
- 신호 계산 단위테스트(임계·경계, near_252 이력요건, 양방향 tilt), 시황 truth-table 분기 테스트, 채점 로직(확정종가 기준·N/A 처리) 테스트, **인덱스 코드 매핑 동일성 테스트**, DART 화이트리스트/무상증자 false-veto 테스트, 백테스트 룩어헤드/풀-재현 가드 테스트, KIS/DART/pykrx 어댑터 목 + **실패/저하 모드 테스트**, 거래캘린더(휴장·조기폐장) 테스트.
- 백테스트는 **순비용 차감 + 베이스라인 대비 incremental rank-IC** 리포트 필수 산출(§7 수용기준).

## 10. 남은 열린 결정 (계획 단계에서 확정)

1. **후보 풀 N**: D-1 거래대금 상위 N 기본 200 — 조정 여부.
2. **KIS 세부 TR/레이트리밋**: 거래대금 랭킹 TR, VI 발동종목 리스트 엔드포인트, 일괄 시세 조회 배치(20 req/s) 확정.
3. **백테스트 기간·생존편향 소스**: 대상 체제 + 상폐/관리 리스트 확보 경로 확정.
4. **운영 환경**: 로컬 PC 상시(15:20 스케줄) vs 별도 호스트 + 알림 채널.
5. **종목 유니버스 정책**: 스팩/리츠/신규상장(이력<252) 취급 세부.
