## 0. 한 줄 요약

매 거래일 **15:20**(정규장 마감 15:30 직전, 특수세션은 마감−10분)에, "종가베팅(신접갈거조재 + 시황 + 수급)" 방법론을 계산 가능한 신호로 번역하여 **다음날 상승 가능성이 높은 종목을 점수 순으로 최대 30개** 추천하는 대시보드. 오버나잇 효과(종가 매수 → 익일 시초 매도)를 캡처한다. 자동 주문은 하지 않으며, **베테랑의 판단을 돕는 의사결정 지원 도구**로 설계한다.

## 2. ⚠️ 전제와 솔직한 리스크 (설계 의사결정의 토대)

적대적 검증에서 확인된 핵심 — **반드시 기대치 설정에 반영한다.**

- **2.1 9개 조건 → 실제 정보축은 3개.** 신·접·갈·조·거·재(긍정부)는 "오늘 거래량 터지며 고가 부근 강마감한 유동성 좋은 종목"이라는 **하나의 베팅의 다른 옷**. 따라서 점수모델은 거를 돌파의 **확인 게이팅(confirmation gating)** 으로 묶어 실효 **3축(① 돌파강도 ② 시황 레짐 ③ 수급)** 만 남긴다(§3.3). (이는 OLS 계수편향 회피가 아니라 분기 시 AND 게이팅 — 그 효과로 §7의 incremental-IC 측정 시 직교화가 필요, §7.)
- **2.2 단기 반전과 싸운다.** 종가베팅 픽은 한국(특히 코스닥)에서 가장 강건한 단기 효과인 **단기 반전(short-term reversal)** 에 역행한다. 인용 근거(52주 신고가=George-Hwang 2004, 거래량 프리미엄=Gervais-Kaniel-Mingelgrin)는 전부 **월간** 횡단면 효과이며 **1일 오버나잇** 이식은 **미검증**. 비용(매도세 0.18%+스프레드+단일가 슬리피지) 차감 후 순엣지 ≤0일 수 있다.
- **2.3 진짜 가치는 종목 찍기가 아니라 리스크 오버레이 + 청산 규율.** 시황 게이트, 희석 veto, 유동성/과열 가드, "보유 금지(익일 시초 청산)" 규율이 좌측꼬리를 줄인다.
- **2.4 신뢰 전 입증 의무(수치 기준).** 어떤 신호든 **walk-forward rank-IC > 0 (t > 2) 이고 순비용 차감 incremental IC(직교화 후, §7)가 오버나잇-ETF/무작위 베이스라인 상회** 전에는 어떤 가중치/배수 범위도 키우지 않는다.

> 결론: "알파 자동 발굴기"가 아니라 "검증된 리스크 필터 위에서 후보를 랭킹·시각화하는 의사결정 지원 도구". 백테스트는 낙관 편향(생존편향·룩어헤드·잠정→확정 갭·데이터소스 결손)을 제거한 뒤에만 신뢰한다.

## 3. 전략 → 계산 가능한 신호 (확정 MVP 셋)

### 3.1 채택/제외 요약

| 원 조건 | 처리 | MVP |
|---|---|---|
| **시황 (지수 5일선 레짐)** | HARD 게이트, 종목 소속시장별 적용 (§3.2-A, 4상태 MECE) | ✅ |
| **신 (52주 신고가 근접도)** | 돌파강도 base (연속 nearness + 돌파 마그니튜드, [근거: 강함(월간)/오버나잇 미검증]) | ✅ |
| **거 (거래대금/거래량)** | 후보 풀 스크리너 + 돌파의 **RVOL 확인 배수**(곱, additive 아님) | ✅ |
| **수급 (외인·기관)** | **D-1 확정치** 양방향 부스터(0.8~1.2x) | ✅ |
| **재 (재료)** | **희석성 공시 = 매수 veto** (긍정 재료 점수화 제외) | ✅(veto만) |
| 위생 가드 | 정적(타입·유동성·관리/경고) + 동적(과열·거래정지) 분리 (§6.2) | ✅ |
| 접/갈 | 신과 collinear·캔들 알파 약함 → **차트 시각만** | 시각만 |
| 조 (조정 기간) | 9파라 과최적 → "충분히 조정됨" **베이스 배지(시각)** | 시각만 |
| 섹터 주도성 | 1일 edge 얇음 → 2차 | 2차 |

> **MVP 가중합에는 s_접·s_갈·s_조가 들어가지 않는다.**

### 3.2 신호별 정의

**A. 시황 — 지수 5일선 레짐 게이트 (4상태 MECE)** [근거: 강건한 추세필터, 알파 아님]
- 코스피·코스닥 **각각** 산출. 입력: 지수 종가(15:20 잠정) + 5일 이평.
- 불리언 **A = (지수 ≥ 5일선)**, **B = (5일선 기울기 > 0; 오늘 5MA > 어제 5MA)**. (당일 등락률 C는 노이즈·중복유발 → 게이트에서 제외, 결정론 확보)
- **총함수(8상태가 아닌 2불리언 4상태로 MECE):**

  | A | B | regime_mult | 의미 |
  |---|---|---|---|
  | T | T | **1.0** | 상승 5MA 위 (정상 상승) |
  | T | F | **0.5** | 5MA 위지만 5MA 꺾임 (약화) |
  | F | T | **0.5** | 상승 5MA 아래 (눌림) |
  | F | F | **0.0** | 하락 5MA 아래 (하락추세 → 중단) |

- **적용**: 종목은 **자신의 상장 시장** 레짐만 사용. 두 시장 모두 0이면 보드 빈 상태 + RISK_OFF 배너(§4.1). 0.5 상태는 **"반-리스크 레짐" 캡션**으로 사유 표기(§4.1).
- 인덱스 코드 충돌 주의: pykrx `1001`=KOSPI/`2001`=KOSDAQ vs KIS `0001`=KOSPI/`1001`=KOSDAQ → **중앙 매핑 + KOSPI 동일성 단위테스트**.

**B. 신 — 52주 신고가 근접도 + 돌파 마그니튜드** [근거: 강함(월간)/오버나잇 미검증 → 잠정 기본 비중 중립]
- `P_now`=15:20 잠정가(KIS). `H_ref`=**전일까지 확정 EOD 일봉 고가** 롤링 최대(룩어헤드 금지). 창: N1=252, N2=60. `near_X = P_now / max(High[t-X..t-1])`.
- **포화 방지(중요)**: near를 1.0에서 하드클립하지 않는다.
  `term(near) = clip((near−0.90)/0.10, 0, 1) + 0.3·clip((near−1.00)/0.05, 0, 1)` (전고점 위 돌파 마그니튜드를 캡 0.3까지 가산).
  `s_신 = 0.7·term(near_252) + 0.3·term(near_60)`.
- **단기 이력(120~251일)**: near_252 불가 → **s_신 = term(near_60) (가중 1.0로 재정규화)**, 라벨 "가용구간 고가". 120일 미만 제외. (251 vs 252 경계 단위테스트)
- **과열 가드**: 당일 등락률 ≥ +20% 또는 상한가/VI → buy emit 금지.
- 돌파 ON CLOSE는 단일가에서 뒤집힐 수 있어 **provisional 플래그**. 보드 상단 동점 정렬은 §3.3 tie-break(거래대금)이 결정 → **상단 내 순서는 신호가 아닌 유동성 기반임을 UI 문서화**.

**C. 거 — RVOL 확인 배수 (MODELED 분모)** [근거: 보통, 비용취약]
- **분모는 MODELED(측정 데이터 아님)**: 15:20 시점 누적거래량 이력이 무료 소스에 없음(pykrx EOD 전용, KIS 분봉 lookback 짧음).
  - 라이브/향후: **매일 15:20 누적거래량 스냅샷을 영구 저장**, ~20세션 축적 후 RVOL 활성화. (또는 측정된 일중 볼륨 프로파일 U자 분율로 EOD 총량 환산)
  - 백테스트: EOD 프록시 + **측정 haircut + 민감도 밴드**, §7에서 거 축의 가중-언락은 축적 전까지 제외.
- `RVOL = (당일 15:20 누적거래량) / (MODELED 15:20-시점 평균)`, `s_거 = clip(log2(RVOL)/log2(3), 0, 1)`.
- **확인 배수**: `rvol_confirm = clip(0.6 + 0.4·s_거, 0.6, 1.0)` (거래량 빈약 돌파 할인 / 충분 시 만점). 거를 가산점으로 두지 않아 신–거 중복 차단.

**D. 수급 — 외인·기관 (D-1 확정, 양방향)** [근거: 진짜·직교, modest]
- pykrx `get_market_net_purchases_of_equities(...)` — **시장별 외인/기관 각각 1회(=총 2회) 일괄** (per-ticker 반복 금지). **순매수거래대금(value) 컬럼** 사용.
- `z = clip((외인+기관 D-1 순매수액) / (20일 평균 거래대금), -1, 1)`.
- **양방향**: `supply_tilt = clip(1.0 + 0.2·z, 0.8, 1.2)` (D-1 순매도는 1.0 미만 패널티).

**E. 재 — 희석 veto (DART 공시)** [근거: veto가 핵심 가치]
- **라이브 윈도우**: 확정 공시 **T-1 15:20 ~ T 15:20**(장중 공시 실시간 사용). post-15:30 당일 공시는 §5 함정 → 추천/채점 제외.
- **희석 화이트리스트**: 유상증자결정, 전환사채권발행결정, 신주인수권부사채권발행결정, 교환사채권발행결정 → `veto=0`. **무상증자/주식배당 제외(non-dilutive, false-veto 금지).**
- corp_code(8)↔티커(6) 매핑 주기 갱신; **미매핑 시 fail-CLOSED**(희석 확인 불가 → 제외, DART-down 정책과 일관).
- **구조적 한계(UI/기대치)**: 15:20 veto는 post-15:20 당일 공시("오버나잇 폭탄")를 못 본다. **익일 아침 재스캔**으로 성과 로그에 플래그(확정된 추천은 불변).

**F. 위생 가드** — **정적(장전 프리페치)**: 우선주/ETF/ETN/SPAC 제외, 20일 평균 거래대금 ≥ **10억(글로벌 유동성 바닥, 1회 정의)**, 관리·투자경고·투자위험·단기과열 지정 제외. **동적(라이브 후)**: 과열(상한가/등락률≥+20%/VI), 15:20 거래정지 제외.

### 3.3 복합 점수 (MVP 실효 공식, 곱셈형 — 실효 3축)
```
# 후보 풀
pool = (pykrx D-1 거래대금 상위 N[기본 200], FINAL·재현가능)   # 장전 사전조회
     ∪ (KIS 라이브 top-30 KOSPI + top-30 KOSDAQ @15:20)        # 당일 신선 돌파 (별도 검증, §7)
# 정적 위생필터 통과분에만 15:20 라이브 시세 조회(~260) → 동적 위생필터

breakout    = s_신                              # MVP: 접/갈/조 가산 없음(시각만)
rvol_confirm= clip(0.6 + 0.4·s_거, 0.6, 1.0)     # 거 = 돌파 확인(곱)
core        = breakout × rvol_confirm × supply_tilt   # 레짐 독립 '품질'(등급 산정 기준, §4.1)
final       = core × regime_mult(0/0.5/1) × veto(0/1) # 게이팅·랭킹 기준
```
- **emit 규칙**: `final > 0` 종목만, **점수 내림차순 상위 30**. 동점 tie-break=D-1 거래대금. 통과 0개면 빈 보드 + 사유 배너.
- **가중치/배수 범위/임계값은 §2.4 / §7 수용기준 충족 전 잠금.**

### 3.4 가격 규칙 (행마다, 결정론적)
- **현재가(표시)** = 15:20 잠정 스냅샷(KIS), "잠정" 워터마크.
- **매수가** = **15:30 종가단일가**(익일 아침 pykrx 확정). 15:20엔 잠정 표시(현재가와 동일값) → **익일 확정 종가로 대체**(스냅샷에 둘 다 보관). **채점·백테스트는 확정 종가 기준**(§4.2).
- **청산(유일한 주 행동 CTA, 채점 대상)** = **익일 시초가**. (09:30 VWAP는 2차 별도 추적 변형 — 주 CTA 아님)
- **목표가/손절가(참고용, '보유 시'·15:20 잠정값으로 스냅샷에 freeze)**:
  - 손절 = `max(매수가 − 1.0·ATR20, 매수가·0.97)` (타이트한 값)
  - 목표 = `직전 전고점 > 매수가 ? min(매수가 + 1.2·ATR20, 직전 전고점) : 매수가 + 1.2·ATR20`
  - k=1.2 고정. ATR20은 일간 레인지라 오버나잇 호라이즌에선 거의 트리거 안 됨 → **참고 정보**(§4.1에서 청산 CTA보다 시각적으로 종속 표기).

## 4. 추천 산출물 & 성과 추적

### 4.1 추천 보드 (메인)
- 점수 내림차순 **최대 30개**. 행: 종목명/코드, 현재가(잠정), 매수가(잠정 종가·익일 확정), **청산=익일 시초(주 CTA, 강조)**, 목표/손절(참고·"보유 시"·종속 표기), 점수·**등급(S/A/B/C)**, 신호 배지(신고가·RVOL·수급·시황·베이스·잠정), 미니 차트.
- **등급 = `core`(레짐 독립) 기준 컷오프**(S≥0.8/A≥0.6/B≥0.4/C>0) → 레짐과 무관하게 "픽 품질"을 의미(레짐은 게이지로 별도). §4.2 점수구간별 적중률도 동일 버킷.
- **상단 강조 top-3~5** + 실행 완료 데스크톱/푸시 알림(10분 결정창 대응). **클라이언트 정렬·필터**(등급/시장/수급배지).
- **빈/저레짐 상태**: regime 0.0 → "오늘은 시황 레짐상 추천 없음" 배너; 0.5 → "반-리스크 레짐(0.5x)" 캡션. 연속 RISK_OFF 시 **스캐너·최근레짐 컨텍스트**를 전면 유지(고장 아님 표시).

### 4.2 성과 추적 (적중률 + 종목별 결과)
- **성공 정의(확정)**: `open[t+1] / close[t] − 1 > 0` — **체결 가능한 15:30 확정 종가 기준**(라이브=백테스트 동일 → 비교가능). 15:20 잠정 기준은 진단용 보조 표기.
- **익일 시가 결측/잠김**: 상한가 잠김(체결불가)/거래정지/데이터없음 → **N/A, 분모 제외**(0점 처리 금지).
- **① 종목별 결과 테이블(사용자 명시 요청, MVP 화면)**: 어제 픽 1행/종목 — 종목명/코드, 등급, 매수가(확정종가), 익일 시가, 익일 시초 수익률, 성공/실패/N·A, DART 오버나잇 재스캔 플래그. 색상 구분, 헤더에 집계 적중률.
- **② 집계**: 적중률 %, 평균 익일 시초 수익률, 누적 곡선, 등급별·레짐별 적중률.
- **콜드스타트**: 누적 픽 < 30이면 "데이터 누적 중" + 최소표본 캡션, 회색 처리.

### 6.1 구성

- **데이터**: `pykrx`(EOD·D-1 수급·지수) + `KIS OpenAPI`(15:20 잠정; 확인된 TR: 거래대금랭킹 `FHPST01710000`, 시세 `FHKST01010100`, 지수 `FHPUP02100000`) + `DART OpenAPI`(희석 veto). `BrokerDataAdapter` 추상화 + 중앙 인덱스/티커 매핑.
- **전송**: **순차 REST 시세 폴링**(~265콜·20req/s ≈ 14s, 15:20–15:30 내). 웹소켓은 41종목 제약으로 스냅샷 부적합 → top-3 실시간 새로고침 옵션만.
- **API(FastAPI)**: `/recommendations/{date}`, `/stock/{code}`, `/performance`, `/backtest`, `/universe`, `/health`.
- **프론트(React+Vite)**: 차트 lightweight-charts/ECharts.
- **저장소**: `state/recommendations/YYYY-MM-DD.json` + **SQLite**.

## 7. 백테스트 & 검증

- **신호가 재현**: 인트라데이 이력 가능 구간은 15:20-등가 스냅샷 재구성. 불가 구간(대부분)은 close[t] 프록시 + **측정 haircut·민감도 밴드**로 EOD→15:20 드리프트 정량화(각주 아님).
- **진입/채점가**: 진입·채점 모두 **확정 종가 close[t]** (라이브 동일) → 비교가능.
- **풀 재구성 & 라이브 top-30 별도검증**: 백테스트는 D-1 상위로 시점기준 재현. **라이브 top-30-only 발생 픽은 백테스트에 부재** → §7 수용기준은 **D-1 서브셋에만 적용**, 라이브 top-30 단독발생 비율을 정량화하고 **fresh-breakout 성분은 별도 paper-forward 검증** 후 신뢰(‘잔여 스큐’로 뭉뚱그리지 않음).
- **incremental-IC 직교화**: 거·수급의 incremental rank-IC는 **신에 대해 직교화한 잔차**로 계산(공유분산에 속지 않도록).
- **DART 시점복원**: list.json은 일자粒度(시각 없음) → 백테스트는 접수시각을 상세계층에서 받아 ≤15:20 필터; 불가하면 **veto를 §7 incremental-IC 회계에서 제외**(point-in-time-backtestable 아님 명시) + 룩어헤드 가드 테스트.
- **생존편향**: point-in-time 유니버스 = pykrx `get_market_ticker_list(date)` + **KRX 상장폐지/정리매매 versioned 스냅샷**. **소스 확보 전 어떤 go/no-go IC도 보고 금지**(주장 게이팅, §10.3).
- **분할 보정**: nearness 비율은 일관보정이면 robust. **RVOL/거래량은 pykrx 보고 거래대금(원가×원량, 분할불변) 컬럼 사용**(adjusted price×volume 재계산 금지).
- **수용 기준(go/no-go)**: §2.4.

## 10. 남은 열린 결정 / 리스크 (계획·스파이크 단계)

1. **후보 풀 N**(기본 200) 조정 여부.
2. **KIS 피저빌리티 스파이크(사전 확정)**: 거래대금 랭킹·시세·지수 TR은 확인됨(§6.1). **남은 확인 = 라이브 VI 발동종목 엔드포인트 존재/커버리지, 인트라데이 거래정지 플래그.** 미존재 시 과열가드는 상한가/등락률≥20%로 폴백. (관리/경고/위험은 준정적 FINAL 프리페치)
3. **백테스트 데이터 소스(리스크)**: ① RVOL **15:20 인트라데이 누적 이력 소스 부재** → 스냅샷 축적 또는 볼륨프로파일; ② **상폐/정리매매 versioned 데이터셋** 확보; ③ DART 접수시각 상세계층 확보. **셋 중 미해결분은 해당 신호를 §7 가중-언락에서 제외.**
4. **운영 환경**: 로컬 PC 상시(15:20) vs 별도 호스트 + 알림 채널.
5. **유니버스 정책**: 스팩/리츠/신규상장(이력<252) 취급 세부.
