# 종가베팅 콘솔 — UI 전면 리디자인 설계 (A2: 터미널 스킨 + 우리말 보이스)

작성: 2026-07-04 · 상태: **사용자 승인됨** (비주얼 컴패니언으로 3방향 → A → A2 순 확정)

## 0. 배경과 목표

현 UI는 기능·정직성은 갖췄으나 "균일한 둥근 카드 그리드 + 이모지 아이콘 + 동일 위계 위젯 나열"로 **AI 생성 사이트의 전형적 관상**이다. 목표는 기능 변경 없이 제품 고유의 얼굴을 입히는 것.

**확정 방향**: A2 "터미널 스킨 + 우리말 보이스" — 블룸버그/HTS 결의 밀도·모노 숫자·헤어라인 시각 시스템에, 지금까지 쌓은 초보자 친화 카피를 그대로 얹는다. 사용자 의사결정 기록:
- 3방향(A 프로터미널 / B 데일리리추얼 / C 에디토리얼) 중 **A 선택** — 단 B를 반복 탐색한 이력을 반영해 B의 핵심 2요소(카운트다운 강조, 성과 잔디 달력)를 터미널 스타일로 이식
- A1(전문용어) vs A2(우리말) 중 **A2 선택**
- 수급 표기는 축약("외+ 기+") 금지 — **"외국인+ 기관−" 풀네임** + 방향색

## 1. 디자인 시스템

### 1.1 색 토큰 (다크 기본 — theme.css `:root` 교체)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg-0` | `#0A0C10` | 앱 배경 |
| `--bg-1` | `#10141B` | 패널 |
| `--bg-2` | `#161B23` | hover/인셋 |
| `--border` | `#232A33` | 헤어라인(모든 위계의 기본 수단) |
| `--text-hi/mid/lo` | `#E8EDF2` / `#8A96A3` / `#5C6773` | 텍스트 3단계 |
| `--up` / `--down` | `#FF5D5D` / `#4D8DFF` | 상승/하락(한국 관례 유지) |
| `--accent` | `#F0B429` | S등급·카운트다운 강조·경고 계열 통합 |
| `--grade-a` | `#3FBF6F` | A등급 |
| 잔디 셀 | 성공 `#A83438` · 실패 `#2B5BB8` · 관망 `#232A33` | 채도 낮춘 관례색 |

라이트 테마: 기존 `:root[data-theme='light']` 오버라이드 구조 유지, "종이 터미널" 톤(밝은 회백 배경 + 동일 위계 원리)으로 값만 교체. 테마 토글·차트 재도색(THEME_EVENT) 계약 불변.

### 1.2 타이포그래피
- 본문: **Pretendard** (`pretendard` npm 패키지 번들, 시스템 폴백 포함 스택)
- 숫자·코드·시각: **JetBrains Mono** (`@fontsource/jetbrains-mono`), `font-variant-numeric: tabular-nums` 전면 적용. 기존 `.mono` 클래스가 이 스택을 갖도록 교체
- 라벨: 한글 소형 라벨(11px 내외, `letter-spacing` 미세 확대). 영문 대문자 라벨 스타일은 쓰지 않음(A2 보이스)

### 1.3 형태·아이콘·모션
- 둥근 카드 폐지: `--radius` 계열 0~2px. 그림자 대신 **헤어라인 디바이더**로 위계. 위젯은 "패널"(제목줄 + 본문, 배경 `--bg-1`, 보더 1px)
- **이모지 전면 제거 → `lucide-react`** (트리셰이킹, 14~16px 스트로크 아이콘)
- 모션 3종만: ① 카운트다운 초침 펄스(15:00 이후) ② 숫자 변경 틱 트랜지션 ③ 스켈레톤 로딩. 그 외 애니메이션 금지

## 2. 화면 설계

### 2.1 보드 (P1)
```
┌─ 상태바 ──────────────────────────────────────────────────────────────┐
│ 종가베팅▮콘솔 · 7/4 금 · 마감까지 10:13 · 코스피 −5.5% · 코스닥 −3.7% │
│ · 오늘 장 분위기: 보수 — 절반만 · 오늘 스캔 전  [후보 가져오기][스캔] │
├─ 탭: 오늘의 추천 | 성과 리포트 (헤어라인 탭, 유지) ────────────────────┤
├─ 메인(좌 ~2/3) ──────────────────────┬─ 레일(우 ~1/3) ────────────────┤
│ 추천 테이블                           │ ① 오늘의 깔때기 200 → 3        │
│  # 등급 종목 현재가* 참고목표 참고손절 │ ② 내 전략의 달력(잔디)          │
│    평소보다거래 수급 재료 담기         │ ③ 기존 위젯 리스킨(시황·신고가·  │
│  TOP3 = 행 좌측 금색 마커+하이라이트   │    공시·수급·캘린더·팔기알림 등) │
│  각주: * 15:20 잠정 · 아침 9~10시 팔기 │                                │
├─ 푸터 라인 ───────────────────────────┴────────────────────────────────┤
│ 커버리지 90.5% · 발행 완료 · 참고용 추천 — 주문은 안 나가요             │
└───────────────────────────────────────────────────────────────────────┘
```
- **상태바**: GlobalHeader를 터미널 상태바로 리스킨. 카운트다운은 15:00 KST부터 `--accent` 강조+펄스. 스캔 상태 칩·정직성 문구·테마 토글·잡 버튼 2종 모두 이 줄에 수용(기존 testid 유지)
- **추천 테이블**: TOP3 카드 컴포넌트 폐지 → 상위 3행 하이라이트로 대체(그 안의 재료 NewsBadge는 테이블 "재료" 컬럼으로 이동). 수급 컬럼 "외국인+ 기관−" 풀네임+방향색. 등급은 색 글자(S 금색/A 녹색/B 파랑/C 회색) — 배지 박스 제거
- **오늘의 깔때기(신규 패널)**: v1은 가용 데이터만 — `후보 N(유니버스 rows) → 추천 M(발행 수)` + 커버리지 + 미발행 사유. 단계별 상세 카운트는 백엔드 확장(백로그 P2 "S6 스캐너 퍼널")과 연동되는 후속 작업으로 명시
- **내 전략의 달력(신규 패널, PerfHeatmap)**: 최근 6~8주 잔디. 데이터는 `/performance` `aggregate.cumulative_curve`의 일별 증분으로 산출(증분>0 성공 빨강, <0 실패 파랑, 곡선에 없는 날 관망 회색). 표본 0이면 정직한 빈 상태. 클릭 → 성과 리포트
- **빈 보드(추천 0건)**: 깔때기 패널이 이유를 설명("신호 통과 0 — 오늘은 관망")하고, 메인 영역은 한 줄 빈 상태 + 다음 행동 안내

### 2.2 종목 상세 (P2)
같은 패널 시스템으로 리스킨. 차트(이미 테마 연동)는 콘솔 팔레트 토큰을 자동 상속. "왜 추천?/뭘 조심?" 2컬럼 구조와 카피 전부 유지.

### 2.3 성과 리포트 (P2)
잔디 달력 월 단위 확대판(주 단위 → 월 그리드) + 기존 집계·표·채점 버튼 유지. 숫자 전부 모노 tabular.

## 3. 데이터·기술 의존성
- 신규 npm: `pretendard`, `@fontsource/jetbrains-mono`, `lucide-react` (전부 로컬 번들 — 외부 CDN 없음)
- 잔디 데이터: 기존 API만 사용(백엔드 무변경). cachedFetch('performance') 재사용
- 깔때기 데이터: 기존 `/universe`(rows 수)·`/recommendations`(발행·추천 수·커버리지)만 사용

## 4. 불변 계약 (리디자인이 깨면 안 되는 것)
1. 모든 기능·데이터 흐름·API 계약
2. 정직성 원칙: 잠정(`*`) 표시, "참고용 추천 — 주문은 안 나가요", 데이터 없으면 정직한 placeholder, 재료 판단은 사람 몫
3. 초보자 카피 전체("평소보다 거래 N배", "다음날 아침 9~10시 팔기" 등)
4. 라이트/다크 토글 + THEME_EVENT 차트 재도색
5. **기존 vitest testid 계약** — 구조 변경 시 testid 이전 필수(예: TOP3 카드 폐지 시 `top3-card` 단언은 행 하이라이트 단언으로 대체하는 테스트 수정 포함)
6. `tsc --noEmit`·빌드·`.js` 0 게이트

## 5. 단계별 실행
| 단계 | 내용 | 산출 |
|---|---|---|
| **P1** | 토큰·폰트·아이콘 기반 교체 + 보드 전체(상태바/테이블/깔때기/잔디/레일 리스킨) | 체감 변화 80% |
| **P2** | 종목 상세·성과 리포트 리스킨(+월 잔디) | 전 화면 일관 |
