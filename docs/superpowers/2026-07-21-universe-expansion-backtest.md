# 유니버스 확대 백테스트 근거 (US-005)

## 결론
스캔 유니버스를 **rank 1–200 → ~600 으로 확대하면 종가매매 수익이 증가**한다.
115거래일(2026-01-26 ~ 2026-07-15) 실측 백테스트에서, rank 201–600 구간의 강한 돌파
종목(base, 60일 고가 3% 이내)은 비용 차감 후 **+0.307%/건**으로 코어 200(+0.344%)과
동급이었다. 601위 이후는 순증이 없어(+0.03%) **상한은 ~600**이 적정하다.

## 데이터 블로커 (실판정창 재현 불가)
전략의 실제 판정 기준은 **익일 09:00–09:20 VWAP**(design §4.2)이나, 이를 다개월
역사로 재현할 **분봉 데이터 소스가 없다**:
- `pykrx` 는 분봉/틱 함수를 제공하지 않는다(일봉 전용 — `dir(pykrx.stock)` 실측 확인).
- KIS 분봉 TR 은 최근 구간만 제공(다개월 역사 조회 불가).

→ 수용기준(US-005)에 따라 **실판정창 walk-forward 는 블로커로 보고**하고, 가장 가까운
**일봉 프록시(D 종가 매수 → D+1 시가 청산, 왕복비용 0.5% 차감)**로 대체한다. 09:00 시가는
09:00–09:20 VWAP 의 합리적 근사다.

## 방법
- 스크립트: `backend/scripts/universe_expansion_backtest.py` (재현 가능, KRX 크리덴셜 필요).
- 전 시장 패널(KOSPI+KOSDAQ ≈ 2,764종목) 115거래일, 60일 워밍업.
- 각 D일: D 거래대금 순위 랭킹 → 60일 고가 근접(near60) + 20일평균거래대금 ≥10억 필터로
  돌파후보 산출 → overnight_ret = D+1_open / D_close − 1.
- `loose` = near60 ≥ 0.90, `base` = near60 ≥ 0.97(실제 발행에 더 가까운 강한 신호).

## 결과 — 밴드별 순수익(after-cost 0.5%)

| 거래대금 순위 | base 순수익 | base 비용초과 승률 | 표본 |
|---|---|---|---|
| 1–200 (현행) | +0.344% | 46.7% | 5,770 |
| 201–400 | +0.384% | 43.8% | 3,724 |
| 401–600 | +0.200% | 41.8% | 2,686 |
| 601–1000 | +0.030% | 35.6% | 3,970 |
| **마진 201–600** | **+0.307%** | 43.0% | 6,410 |

- **컷 밖 = 동전던지기** 라던 이전 저신뢰(12일) 우려는 표본 115일에서 **기각**됨.
- 시장별: KOSDAQ 201–600 +0.096% vs KOSPI 201–600 −0.049%(loose) — 확대 알파는
  저커버리지 **코스닥에 집중**(현재 코스닥은 1,821종목 중 68개=3.7%만 스캔).

## 한계
- 실판정창(VWAP) 아닌 시가 프록시. full score(수급·레짐·veto) 미반영 —
  **밴드 상대 비교는 견고**하나 절대 픽 수는 과대평가.
- 측정 구간 2026 H1(레짐 편향 가능). 실판정 검증은 분봉 데이터 확보 후 재수행 권장.

## 함의 (다음 단계)
- 프로덕션 유니버스 기본값 변경은 **위생 실배선(US-002)** 완료 + 위 한계 인지 하에
  `CBR_UNIVERSE_N`(US-003) 으로 단계적 상향(예: 300→500). 급격한 전 종목 확대는 불필요.

## 확대 도입 검증 (US-004)

### 15:20 시간 예산 — 확대해도 창 안에 들어온다 (US-001 병렬화 이후)
- KIS 호출은 pace 락으로 초당 ~20건(rate_per_sec=20)에 수렴. 15:20 KIS 부하 = 시세 N +
  기본정보 N(위생) = 2N 콜, 두 병렬 패스로 **≈ 2N/20 초**. DART veto 는 US-001 벌크화로
  종목수 무관(페이지 몇 콜).
- N=200 → ≈ 20초, N=600 → ≈ 60초. 창: 15:18 기동, 15:20 스냅샷(120초 예산), 15:30
  마감(720초). **N=600 도 스냅샷 예산(120초) 안**에 든다.
- 대조(병렬화 전): N=200 직렬이 실측 125~221초로 이미 15:20 초과였다. US-001 로
  ~15-20배 개선(직렬 왕복 0.6~1.1초/종목 → 병렬 20/초). **확대의 물리적 전제 충족.**

### 커버리지 회귀 없음 (테스트로 고정)
- `backend/tests/engine/test_pipeline.py`:
  `test_pipeline_scales_to_expanded_universe`(N=600 정상 발행, MAX_EMIT 상한, 랭킹),
  `test_pipeline_expanded_universe_partial_coverage_still_publishes`(꼬리 실패해도
  커버리지 ≥0.70 이면 보드 유지). 확대 시 '꼬리 실패 → 보드 전체 LOW_COVERAGE 차단'
  회귀를 가드한다.
- 안전 방벽: US-002 로 우선주/관리/경고 종목이 정적위생에서 실제 제외되므로, 확대해도
  쓰레기 종목이 추천에 섞이지 않는다.

### 롤아웃 게이팅 (프로덕션 기본값은 200 유지)
- 기본값은 `CBR_UNIVERSE_N` 미설정 시 **200 유지**(US-003). 확대는 env 로 opt-in,
  **단계적 상향 권장(300→500→600)**.
- 실판정창(VWAP) full-score walk-forward 검증(US-005)은 분봉 데이터 확보 후 재수행 —
  그전까지 일봉 프록시 근거로 단계적 확대. 급격한 전 종목 스캔은 불필요(601+ 순증 없음).
