# 종가베팅 추천 대시보드 — UI 설계 제안서

작성일 2026-07-01 · 대상 화면 React/Vite (Board / StockDetail / Performance / RegimeGauge / Scanner)
사용 맥락 15:20~15:30 10분 결정창 · 데이터 한계 현재가 15:20 잠정 · 수급 D-1 확정 · 호가/체결/분봉 미연동

---

## 0. 결론 3줄

1. 지금 필요한 건 예쁜 차트가 아니라 **색으로 갈린 위계(등급/방향/리스크)** 와 **잠정 vs 확정 정직성 배지**다. 이 둘이 없으면 10분 결정창에서 못 쓴다.
2. 코드 대부분(등급 클래스 `grade-S`, `badge-*`, `exit-cta`)이 이미 마크업에 있다. **P0는 CSS 파일 하나(다크 테마 + 위계)로 체감 80%가 바뀐다.**
3. 5인 패널의 공통 1순위는 완전히 일치한다 → **등급 색 위계 + 데이터 신선도 배지 + 리스크 적색 플래그.** 여기부터 친다.

## 1. 핵심 인사이트 — 공통 요구 vs 상충과 해소

### 1-1. 5인 전원 공통 요구 (이견 없음 = 무조건 P0)

| # | 공통 요구 | 근거 (패널 다수 인용) |
|---|-----------|----------------------|
| C1 | 등급 S/A/B/C **색·배경 위계** | 30/20/10/5년차 전원 "다 같은 검정 글씨라 S와 C가 안 갈린다" |
| C2 | **데이터 신선도 배지** (15:20 잠정 vs D-1 확정) | 전원 top-priority. "잠정을 확정으로 오해하면 사고" |
| C3 | **리스크 적색 플래그** (희석 veto·관리종목·익일 이벤트) | "링크 수준으로 묻힌다. 적색이 우선 노출돼야" |
| C4 | 상승=빨강 / 하락=파랑 **한국 관례 방향색** | 20/10년차 명시 |
| C5 | **기대수익 + 리스크 병기** (반쪽 정보 금지) | 30년차 "먹을 폭만 있고 깨질 폭이 없다" |
| C6 | 5승수(신·거·시황·수급·재) **시각 분해** | 전원 "왜 이 종목인지가 점수 하나로 뭉쳐 안 보인다" |

### 1-2. 페르소나별 상충과 해소안

| 상충 축 | A 진영 | B 진영 | 해소안 (결정) |
|---------|--------|--------|--------------|
| **정보 밀도** | 10년차/20년차: "빽빽하게, 컬럼 많이" | 5년차(개미): "폰으로 5초, top3만" | **밀도 토글**: 기본=Compact(빽빽 테이블), 헤더에 `요약` 스위치로 top3 카드 뷰 전환. 반응형으로 모바일은 자동 카드. 한 데이터·두 렌더. |
| **친절함 vs 전문성** | 개미: 툴팁·설명 필요 | 베테랑: "설명 방해, 정보만" | **점진적 노출**: 기본은 정보만. 용어에 `?` 호버 툴팁(전문가는 무시, 초보는 호버). 최초 1회 온보딩 코치마크는 dismiss 후 영구 숨김. |
| **화면 이동** | 개미/10년차: 모달 퀵카드로 제자리 판단 | 30년차: 상세 페이지에서 깊게 검증 | **둘 다**: Space=사이드 퀵카드(스캔용), Enter/클릭=풀 상세 페이지(검증용). 같은 컴포넌트 재사용. |
| **결론 강제** | 개미: "GO/보수/스킵 한 줄" | 베테랑: "판단은 내가, 데이터만 줘" | 결론은 **권고(추천 강도)** 로 표기하되 강제 안 함. "시황 0.5x → 사이즈 절반 권장" 배너, 최종 판단은 사용자. |
| **대외 리스크** | 30/20년차: 선물·환율·VIX 필수 | 데이터 현실: 미연동 | 패널 자리는 만들되 **"미연동" 회색 플레이스홀더**로 정직하게. 연동 로드맵 스위치 자리 확보. |

> 핵심 설계 철학: **하나의 데이터, 밀도 토글로 두 관객(개미/전문가)을 동시 만족.** 별도 앱을 만들지 않는다.

### 2-2. 타이포그래피

| 역할 | 스펙 |
|------|------|
| 숫자 (가격·수익률·score) | `ui-monospace, "Roboto Mono", monospace` — **탭 정렬 필수**, 자릿수 흔들림 방지 (`font-variant-numeric: tabular-nums`) |
| 종목명·본문 | `Pretendard, -apple-system, sans-serif` |
| 스케일 | 결정 핵심(청산가·기대수익) 18px/700 · 일반 셀 14px/500 · 라벨 12px/500 · 주석 11px |
| 위계 규칙 | **한 행에서 굵고 큰 건 딱 2개**(등급 배지 + 기대수익). 나머지는 mid 톤. |

### 2-3. 정보 밀도

- 테이블 행 높이 **36px**(Compact) — 30개가 스크롤 없이 한 화면에 최대한.
- 셀 패딩 좌우 10px / 상하 6px.
- 밀도 토글: `Compact`(기본) / `Cozy`(44px, 초보·모바일).

### 2-4. 10분 결정창 UX 5원칙

| 원칙 | 실행 |
|------|------|
| P1 **10초 스캔** | 등급 색 + top3 카드 분리 + 방향색으로 시선 유도. "무엇부터 볼지"가 색으로 결정. |
| P2 **정직성 상시 노출** | 헤더 고정 배너 "15:20 잠정 · 수급 D-1 · 체결 미연동" + 갱신 타임스탬프 + 마감 카운트다운. |
| P3 **손 안 떼기(키보드)** | `1~9`=담기, `↑↓`=행 이동, `Space`=퀵카드, `S`=S/A필터, `F`=정렬 순환. |
| P4 **결론으로 시선 수렴** | 상단 퀵 요약 헤더에 오늘 판정(GO/보수/스킵) 한 줄. |
| P5 **리스크 우선** | 적색 플래그는 항상 행 좌측 고정 컬럼, 스크롤·정렬과 무관하게 먼저 보이게. |

## 5. 개미(초보) 배려 — 어디에 어떻게

원칙: **전문가를 방해하지 않는 점진적 노출.** 기본 화면은 정보만, 설명은 호버·최초 1회로.

| 위치 | 방식 | 내용 |
|------|------|------|
| 용어 옆 `?` | 호버 툴팁(전문가는 무시) | "RVOL = 평소 대비 오늘 거래량 배수. 3배↑면 관심 몰림", "오전VWAP = 다음날 아침 거래량가중평균가, 청산 기준" |
| 등급 배지 호버 | 툴팁 | "S = 신·거·시황·수급 4요소 모두 강함. 확신 최상" |
| 잠정 배지 호버 | 툴팁 | "15:20 값이라 마감(15:30)에 바뀔 수 있음. 확정 아님" |
| 최초 진입 1회 | 코치마크(dismiss 후 영구 숨김) | "① 색으로 등급 확인 ② 잠정 배지 주의 ③ 담기 1~9" |
| 리스크 플래그 | 상시 적색 + 호버 상세 | "⚠ 희석 = 유상증자/CB로 주식수 증가 예정. 하락 위험, 추천서 제외 권고" |
| 시황 배너 | 상시 한 줄 결론 | "오늘 보수적(0.5x): 시장 눌림. 평소보다 종목수·비중 줄이세요" |
| 성과 신뢰구간 | 주석 | "표본 적으면(±폭 넓음) 이 숫자 과신 금지" |
| 빈 상태·처음 | 온보딩 카드 | "이 대시보드는 추천만 합니다(주문 없음). 15:20 잠정 데이터 기반." |

리스크 경고 배치 규칙: **경고는 항상 초록/빨강 대비로 왼쪽·위에.** 사용자가 스크롤하거나 정렬을 바꿔도 리스크 `⚑` 컬럼은 고정. "좋은 이유"보다 "위험한 이유"를 먼저 보이게 해 오버나잇 사고를 막는다.
